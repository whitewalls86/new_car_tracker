How data flows

Ingest
Scraper → MinIO

Raw HTML artifacts written to MinIO object storage, partitioned by date and artifact type.

Bronze
Processing → Postgres

Artifacts claimed from the queue, parsed, and written to staging event tables in Postgres.

Silver
Archiver → Parquet

Archiver bulk-flushes staging events to Parquet files on MinIO, partitioned by source and date.

Mart
dbt + DuckDB

dbt reads Parquet from MinIO and Postgres exports, producing silver and mart layers in DuckDB.

Serving
Dashboard

Streamlit queries the DuckDB mart layer directly for price history, deal scores, and trends.

Architecture at a glance

Click any card to expand.

Scraper FastAPI

Fetches SRP and detail pages with Chrome TLS fingerprinting; writes compressed HTML to MinIO.
  • Uses curl_cffi to mimic a real Chrome TLS handshake, bypassing basic bot detection
  • Automatic fallback to FlareSolverr for Cloudflare JS challenge pages
  • HTML artifacts GZIP-compressed before writing to MinIO with structured key paths
  • Process-wide credential cache with 25-min TTL; auto re-bootstraps on 403
  • Airflow triggers scrape rotation every 15 minutes across all tracked make/model pairs
Processing FastAPI

Claims artifacts from the queue, parses HTML, and writes price observations via staging event tables.
  • Polls ops.artifacts_queue using atomic FOR UPDATE SKIP LOCKED claims to prevent duplicate processing
  • BeautifulSoup extracts listing fields: price, mileage, VIN, dealer, trim, and more
  • Writes to staging._events tables first — never directly to HOT tables
  • Idempotent by VIN: duplicate SRP hits produce a price observation, not a duplicate vehicle record
Ops FastAPI

Admin UI, deploy coordination, claim management, and auth forward-proxy endpoint.
  • Admin routes for monitoring queue depth, claim state, and per-service health
  • forward_auth endpoint: Caddy calls this for every protected route to validate Google OAuth tokens and DB role
  • DB-backed user access request flow with role assignment
  • Deploy coordination endpoint triggers zero-downtime container restarts
Archiver FastAPI

Bulk-flushes staging event buffers to MinIO Parquet; sweeps completed queue rows.
  • Triggered by Airflow on a configurable schedule (default: hourly)
  • Reads staging._events tables in batches and writes Parquet to MinIO partitioned by source/YYYY-MM-DD
  • After a successful flush, promotes events to HOT tables and marks queue rows complete
  • Separate sweep job removes completed queue entries beyond the retention window
dbt Runner dbt + DuckDB

Runs dbt transformations against DuckDB — reads Parquet from MinIO directly, no separate warehouse needed.
  • DuckDB reads Parquet directly from MinIO via the httpfs extension — no ETL step needed
  • Model layers: staging → silver (cleaned, typed) → mart (deal scores, price windows, rolling averages)
  • blocked_cooldown model implements exponential backoff logic for 403'd listings in pure SQL — queryable and testable
  • Full dbt test suite runs on each pipeline execution via Airflow
Airflow Airflow

Schedules all DAGs — scrape rotation, artifact processing, staging flush, queue cleanup, and orphan checks.
  • Twelve active DAGs covering scrape rotation, detail scraping, artifact processing, staging flush, dbt builds, orphan claim expiry, queue cleanup, Parquet cleanup, and deployment coordination
  • Task-level retries and alerting; no shared state between tasks
  • DAG changes deploy via git pull — no container restarts required
  • Replaced the previous n8n workflow engine, which was decommissioned
Dashboard Streamlit

Interactive price-history and deal-score views backed by the DuckDB mart layer.
  • Queries DuckDB mart layer directly — sub-second response on 100K+ observations
  • Price history charts per listing with deal score overlay and days-on-market trends
  • Make/model/trim filtering, price drop alerts, and rolling average comparisons
  • Access-gated via Caddy OAuth — same auth layer as the rest of the stack
MinIO S3-compatible

Object store for raw HTML artifacts and Parquet silver observations; queryable directly by DuckDB.
  • S3-compatible object store running on the same host as Postgres
  • Two buckets: raw (GZIP HTML artifacts) and silver (Parquet price observations)
  • DuckDB reads silver Parquet directly via httpfs + S3 credentials — no separate data warehouse
  • Raw artifacts retained for 30 days; Parquet files retained indefinitely
Caddy Reverse Proxy

Reverse proxy with TLS termination, OAuth2 Google auth, and DB-backed role checks.
  • Automatic TLS cert provisioning via Let's Encrypt
  • forward_auth middleware calls the Ops service for every protected route — validates Google OAuth token and DB role
  • Routes: / → Dashboard, /airflow → Airflow, /grafana → Grafana, /info → public (no auth)
  • Single Caddyfile defines the entire routing and auth layer
Grafana Observability

Operational dashboards from Prometheus metrics (Airflow, Postgres, MinIO, node).
  • Scrapes Prometheus metrics from Airflow, Postgres (postgres_exporter), MinIO, and node_exporter
  • Dashboards cover pipeline throughput, queue depth, processing lag, and system resources
  • Deployed at /grafana behind the same Caddy auth layer
  • Prometheus scrape targets for all services added via V033 migration

Technical highlights

Medallion architecture across two storage systems

Each layer uses the right engine for its access pattern: Postgres for low-latency operational reads, Parquet on MinIO for durable event history, DuckDB for analytical queries. One tool was never going to do all three well.

Event sourcing via staging flush

Price observations write to staging._events tables first; the archiver bulk-flushes them to HOT tables and Parquet on a schedule. Decouples write throughput from read consistency and enables bulk-insert optimization.

Atomic claim pattern

No Redis, no SQS — Postgres ON CONFLICT DO UPDATE handles distributed claim locking natively, keeping the stack simple without sacrificing correctness. An Airflow orphan-checker DAG expires stale claims left by crashed containers.

dbt backoff model

403'd listings are tracked in blocked_cooldown; exponential backoff logic lives entirely in a dbt model — queryable and testable without touching application code.

Versioned schema migrations

Every schema change is a reviewed PR — no ad-hoc ALTER TABLE in production. 37 Flyway migrations covering table creation, index additions, HOT table promotion, role grants, and legacy drops, applied automatically on deploy.

Three-layer test strategy

SQL smoke tests (Layer 1) validate schema correctness before dbt runs. dbt logic tests (Layer 2) seed known inputs and assert outputs against a CI schema. API integration tests (Layer 3) verify full request paths against a real Postgres instance — no mocks where real DB behavior matters. 971 tests total, enforced on every commit.

Drain-based zero-downtime deploys

Each service exposes a /ready endpoint. A deploy_intent table signals pending deployments; Airflow sensors block new DAG runs until all in-flight work drains. Container restarts happen only between runs — never mid-pipeline.

Anti-detection resilience

curl_cffi with Chrome TLS fingerprinting + FlareSolverr for Cloudflare JS challenges. Process-wide credential cache with 25-min TTL and automatic re-bootstrap on 403.
